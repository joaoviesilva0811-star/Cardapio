Cole isto em: Firebase Console → Firestore Database → aba "Regras" → Publicar.

Essas regras deixam qualquer visitante LER o cardápio (categorias, atributos,
produtos), mas só permitem CRIAR/EDITAR/EXCLUIR para quem estiver logado
(ou seja, através do painel admin, depois de entrar com email/senha
cadastrados em Authentication → Users).

------------------------------------------------------------------

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    match /categorias/{docId} {
      allow read: if true;
      allow write: if request.auth != null;
    }

    match /atributos/{docId} {
      allow read: if true;
      allow write: if request.auth != null;
    }

    match /produtos/{docId} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}

------------------------------------------------------------------

Depois de colar e publicar, teste:
1. Abra o site (index.html) sem estar logado — o cardápio deve carregar normalmente.
2. Tente entrar no painel admin sem estar logado — deve te mandar para login.html.
3. Faça login com um usuário criado em Authentication → Users e cadastre um
   produto — ele deve aparecer tanto no seu navegador quanto no de outra pessoa
   acessando o mesmo link.
